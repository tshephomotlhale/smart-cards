# Smart Patient Card System

> Botswana National Open Data Hackathon 2026 — Health Track
> Patient Flow + Supply Chain Management for public health facilities

## What It Does

Patients tap a smart card at any clinic or hospital to check in. The system queues them by urgency, lets them submit symptoms via mobile app or USSD, and notifies clinical staff in real time as the patient moves through triage → consultation → pharmacy → discharge.

Staff dashboards show live queue state, wait times, digital adoption rates, and medicine stock health across all facilities.

---

## Tech Stack

| Layer | Technology |
|---|---|
| Backend API | FastAPI (Python 3.12, async) |
| Database | PostgreSQL 16 |
| Cache / Queue | Redis 7 |
| Containerisation | Docker + Docker Compose |
| Auth | JWT + bcrypt (6-role RBAC) |
| SMS / USSD | Africa's Talking |
| Real-time | Server-Sent Events (SSE) |
| Frontend | Next.js (coming soon) |

## Quick Start

**Requirements**: Docker Desktop running

```bash
# Clone
git clone https://github.com/tshephomotlhale/smart-cards.git
cd smart-cards

# Copy env and fill in your credentials
cp backend/.env.example backend/.env

# Start everything (builds, migrates, seeds, prints URLs)
bash start.sh
```

API will be available at **http://localhost:8000**
Swagger docs at **http://localhost:8000/docs**

Default admin login: `admin@smartcards.bw` / `Admin1234!`

## Environment Variables

Copy `backend/.env.example` to `backend/.env` and set:

```env
SECRET_KEY=          # auto-generated by start.sh
DATABASE_URL=        # pre-filled for local Docker
REDIS_URL=           # pre-filled for local Docker
AT_USERNAME=         # Africa's Talking username
AT_API_KEY=          # Africa's Talking API key
AT_SENDER_ID=        # SMS sender name (e.g. SmartCard)
ALLOWED_ORIGINS=     # frontend URL (e.g. http://localhost:3000)
```

## API Overview

### Auth
| Method | Endpoint | Description |
|---|---|---|
| POST | `/auth/login` | Login → JWT token |
| POST | `/auth/register` | Register staff (admin only) |

### Patients
| Method | Endpoint | Description |
|---|---|---|
| POST | `/patients/` | Register new patient |
| GET | `/patients/{id}` | Get patient details |
| POST | `/patients/{id}/cards` | Issue smart card |

### Visits
| Method | Endpoint | Description |
|---|---|---|
| POST | `/visits/arrive` | Check in via card tap |
| POST | `/visits/walkin` | Check in walk-in patient |
| POST | `/visits/symptoms` | Submit symptom answers |
| POST | `/visits/triage` | Nurse triage + urgency |
| GET | `/visits/{id}` | Get visit details |

### Queue
| Method | Endpoint | Description |
|---|---|---|
| GET | `/queue/{facility_id}` | Full queue snapshot |
| GET | `/queue/{facility_id}/position/{visit_id}` | Patient's position + wait |

### Pharmacy
| Method | Endpoint | Description |
|---|---|---|
| POST | `/pharmacy/prescriptions` | Create prescription |
| POST | `/pharmacy/dispense` | Dispense + deduct stock |
| GET | `/pharmacy/stock/{facility_id}` | Current stock levels |
| GET | `/pharmacy/demand/{facility_id}` | Pending prescription demand |

### Analytics
| Method | Endpoint | Description |
|---|---|---|
| GET | `/analytics/dashboard/{facility_id}` | All KPIs in one call |
| GET | `/analytics/throughput/{facility_id}` | Arrivals / active / discharged |
| GET | `/analytics/funnel/{facility_id}` | Patient drop-off per stage |
| GET | `/analytics/wait-times/{facility_id}` | Avg/max/min triage wait |
| GET | `/analytics/adoption/{facility_id}` | Digital intake rate by channel |
| GET | `/analytics/stock/{facility_id}` | Stock health per medicine |

### Real-time
| Method | Endpoint | Description |
|---|---|---|
| GET | `/events/{facility_id}/{role}` | SSE stream for dashboard role |

### USSD
| Method | Endpoint | Description |
|---|---|---|
| POST | `/ussd` | Africa's Talking webhook |

## Patient Flow

```
TAP CARD / WALK-IN
      │
      ▼
   ARRIVED ──────► Redis priority queue
      │
      ▼
SYMPTOMS SUBMITTED (app / USSD / kiosk)
      │
      ▼
   TRIAGED ────────► urgency score → re-ranked in queue
      │
      ▼
IN CONSULTATION
      │
      ▼
  PRESCRIBED ──────► pharmacy notified (SSE)
      │
      ▼
 AT PHARMACY ────────► stock deducted, patient SMS sent
      │
      ▼
  DISCHARGED
```
